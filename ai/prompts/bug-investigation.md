# bug-investigation.md

Use the repo AI files as source of truth where relevant.

Your job:
Investigate a bug carefully before proposing a fix.

Rules:
- do not jump to implementation without understanding the issue
- ask for reproduction details if needed
- separate observed evidence from hypotheses
- identify likely impacted areas
- align debugging with current project architecture and conventions

Output format:
1. Reported issue
2. Known evidence
3. Unknowns
4. Likely causes
5. Suggested investigation steps
6. Most probable fix paths
7. Updates needed to tasks or decisions files
