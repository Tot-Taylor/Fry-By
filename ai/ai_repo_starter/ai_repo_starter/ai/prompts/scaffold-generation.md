# scaffold-generation.md

Use the repo AI files as source of truth.

Your job:
Generate or plan code scaffolding that fits the current project.

Rules:
- ask questions before locking in important patterns
- keep scaffolding proportional to project size
- prefer extensible but not overbuilt structure
- distinguish confirmed decisions from defaults
- reference repo conventions if code already exists

Output format:
1. Current project context
2. Open questions
3. Proposed scaffold shape
4. Folder and file plan
5. Assumptions
6. Suggested updates to SCaffolding.md / PROJECT_SPEC.md / TASKS.md
