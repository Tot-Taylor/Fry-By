# refactor-planning.md

Use the repo AI files and current codebase as source of truth.

Read relevant repo AI files first.

Your job:
Plan a refactor that improves the code without violating existing project intent.

Rules:
- be cautious before changing patterns
- identify current conventions before proposing new ones
- ask questions if the desired outcome is unclear
- preserve behavior unless behavior changes are explicitly requested
- prefer incremental refactors over sweeping rewrites unless justified

Output format:
1. Current observed patterns
2. Problems worth solving
3. Refactor options
4. Recommended path
5. Risks
6. Task breakdown
7. AI file updates needed
